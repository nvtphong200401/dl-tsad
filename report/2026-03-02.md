## Architecture Overview

The dl-tsad system uses a modular 4-step pipeline for time series anomaly detection, combining STL decomposition with LLM-based range detection.

```
Raw Series (T, 1)
    |
[Step 1: STLProcessor] ‚îÄ‚îÄ‚îÄ‚îÄ Deseasonalized residual ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ> [Step 3: LLM]
    |                   ‚îÄ‚îÄ‚îÄ‚îÄ In-window forecasts ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ> [Step 2]
    v
[Step 2: EvidenceBasedDetection] ‚îÄ‚îÄ Evidence scores ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ> [Step 3: hints]
    |
    v
[Step 3: LLMRangeDetectionScoring]
    |   Strategy 1: CUSUM pre-check (skip normal series)
    |   Strategy 2: Running mean validation (reject FP ranges)
    |   Strategy 3: Range refinement (grow from peak rmdev)
    v
[Step 4: PostProcessor]
    |   FixedThreshold(0.5) + filter + merge
    v
Binary Predictions (T,)
```

## Step 1: Data Processing ‚Äî STLProcessor

**File**: `src/pipeline/step1_stl_processor.py`
**Config type**: `STLProcessor`

### Purpose
Transforms raw time series into normalized windows and generates two outputs for downstream steps:
1. **In-window forecasts** for evidence extraction (Step 2)
2. **Deseasonalized residual** for LLM range detection (Step 3)

### Process

**Training (`fit_transform`)**:
1. Creates sliding windows from training data (N windows, size W, stride S)
2. Reconstructs full series from overlapping windows
3. Auto-detects seasonal period via ACF peak detection
4. Runs STL decomposition ‚Üí extracts seasonal pattern + linear trend model
5. Computes forecast error distribution on training data (for confidence bands)
6. Stores training statistics (mean, std, quantiles) for evidence extraction

**Inference (`transform`)**:
1. Normalizes test windows using training statistics
2. Runs STL on test series ‚Üí stores **residual** as deseasonalized series
   - Key: uses test STL for deseasonalizing (removes both trend AND seasonal)
   - This ensures no edge drift artifacts
3. Builds **expected series** from training model (trend + seasonal) ‚Äî NOT test STL
   - Key: training model doesn't absorb test anomalies
4. Generates per-window in-window forecasts with bootstrap confidence bands

### Key Design Decisions
- **In-window forecasting** (not forward-looking): forecast length == window size, compares expected vs actual within the same window. Works at all positions including series edges.
- **Training model for forecasts, test STL for deseasonalizing**: Different STL runs serve different purposes. Training model provides the "expected" baseline for evidence. Test STL provides clean residuals for the LLM.
- **Forecast error from training** (not STL residuals): The actual error between training model and training data captures the real mismatch the model will have on test data, producing realistic confidence bands.

## Step 2: Detection ‚Äî EvidenceBasedDetection

**File**: `src/pipeline/step2_detection.py`
**Config type**: `EvidenceBasedDetection`

### Purpose
Extracts 13 statistical anomaly metrics per window, providing both a scalar anomaly score and rich evidence dictionaries for downstream LLM reasoning.

### Evidence Categories

| Category | Metrics | What it detects |
|----------|---------|-----------------|
| **Forecast-based** (5) | MAE, MSE, MAPE, quantile violations, surprise score | Point anomalies, level shifts, forecast deviation |
| **Statistical tests** (3) | Z-score, Grubbs test, CUSUM | Outliers, single spikes, mean shifts |
| **Distribution** (2) | KL divergence, Wasserstein distance | Collective shifts, distribution changes |
| **Pattern** (3) | ACF break, volatility ratio, trend break | Frequency changes, variance spikes, slope changes |

### Process
1. Receives forecast context from Step 1 (in-window forecasts + training statistics)
2. Auto-detects forecast type: `len(forecast) == window_size` ‚Üí in-window comparison; otherwise ‚Üí forward comparison
3. For each window: runs all enabled extractors, produces evidence dict
4. Aggregates 13 metrics into scalar score via weighted sum normalized by training P95 baselines

### Output
- Subsequence scores: (N,) ‚Äî one scalar per window
- Evidence dicts: (N dicts) ‚Äî full metric details per window, used by LLM scoring as "evidence hints"

## Step 3: Scoring ‚Äî LLMRangeDetectionScoring

**File**: `src/pipeline/step3_scoring.py` (class `LLMRangeDetectionScoring`)
**Config type**: `LLMRangeDetectionScoring`

### Purpose
Uses an LLM (GPT-4o Mini via Azure) to detect anomaly ranges directly from the deseasonalized series. Includes 3 post-processing strategies to reduce false positives and improve range accuracy.

### Multi-Channel Prompt

The LLM receives 1000 data points with 3 channels each:
```
(index, R, C, M)
```

| Channel | Signal | What it shows | Scaling |
|---------|--------|---------------|---------|
| R | STL Residual | Point anomalies (spikes/dips) | Min-max [0, 10] |
| C | Bidirectional CUSUM | Anomaly onset (transition detection) | 10*std reference |
| M | Running Mean Deviation | Full anomaly extent (sustained shift) | Center 5.0, +/-2.5 per std |

**CUSUM parameters**: slack = 3.0 * std (high slack to suppress periodic residual oscillations, ACF~0.99 at lag 1)

**LLM instruction**: Use C to find where anomaly STARTS, use M to find where it ENDS.

### Strategy 1: CUSUM Pre-check

**Purpose**: Skip LLM API call entirely for normal series (saves cost + eliminates FPs)

- Computes `cusum_max_scaled = cusum.max() / (10 * std) * 10`
- If `cusum_max_scaled < 2.0` ‚Üí return zeros immediately
- Normal series: CUSUM = 0.0 (perfect separation)
- Anomalous series: CUSUM > 2.0

**Impact**: Skips 230/400 normal trend series ‚Üí 0 FPs from normal series, ~57% API cost reduction.

### Strategy 2: Running Mean Validation

**Purpose**: Reject LLM-returned ranges that don't have sustained running mean deviation

- For each LLM range: compute `mean(|rmdev|) / std` at the range
- If `rmdev < 0.5*std` ‚Üí REJECT (false positive from LLM hallucination)
- Running mean (window=50) sustains during anomalies unlike CUSUM which resets

### Strategy 3: Range Refinement

**Purpose**: Both EXTEND narrow onset-only ranges and TRIM overly wide ranges

The LLM consistently flags the anomaly **onset** (CUSUM spike region) rather than the full extent. The refinement uses wide running mean (window=100) to find the true boundaries:

1. Find **peak rmdev point** within ¬±50 of the LLM range
2. Grow outward from peak while `rmdev > 0.5*std`
3. This converges to the same refined range regardless of whether the LLM returned a narrow or wide range

**Key insight**: CUSUM detects the transition then resets. Running mean sustains during the anomaly. Use each where it's strongest.

### Process Flow
```
Deseasonalized series
    |
    v
[CUSUM pre-check] ‚îÄ‚îÄ flat? ‚îÄ‚îÄ ‚Üí return zeros (no anomaly)
    |
    | signal detected
    v
```

## Step 4: Post-Processing

**File**: `src/pipeline/step4_postprocessing.py`
**Config type**: `FixedThreshold` + filter + merge

### Purpose
Converts binary point scores to final predictions with cleanup.

### Process
1. **FixedThreshold(0.5)**: Designed for binary LLM output (0.0 or 1.0). Percentile thresholds fail on binary scores.
2. **Filter short anomalies**: Remove segments < 3 points (noise)
3. **Merge close anomalies**: Join segments separated by ‚â§ 5 points (fragmentation)

## Evaluation

### Metrics
- **Point-wise F1**: Standard precision/recall on individual points
- **PA-F1**: Point-adjusted F1 ‚Äî if any predicted point overlaps a ground truth segment, count as TP
- **VUS-PR**: Volume under PR curve ‚Äî threshold-independent metric
- **Precision/Recall**: True negatives (no anomaly, no detection) count as precision=1, recall=1

## Synthetic Data Characteristics (Trend Category)

- **400 eval series**, 1000 points each, univariate sine wave
- **230 series** have 0 anomalies (57.5%)
- **170 series** have anomalies at [800, end), length 30-200 points
- Anomalies are **subtle trend shifts**: mean shift ~0.1 on [-1, 1] scale (0.6-1.4œÉ in residual)
- The anomaly is a smooth phase/offset change ‚Äî no visible spike in the raw signal

---

## Known Limitations

1. **Long subtle anomalies** (200+ points, <1œÉ shift): Running mean oscillates below threshold before covering the full extent. Only ~60% of very long anomalies are captured.
2. **LLM onset bias**: The LLM consistently flags the CUSUM transition region rather than the sustained anomaly. Strategy 3 (refinement) mitigates this but depends on running mean signal strength.
3. **Sine wave periodicity in residual**: STL residual retains ACF~0.99 at lag 1. CUSUM slack must be 3.0*std to avoid false triggers from normal half-cycles.
4. **Single-series evaluation**: The benchmark concatenates all series for global metrics. Per-series averaging would better reflect the 230 true-negative series.
